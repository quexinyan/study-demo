数据分类:
   1.结构化数据 数据库数据 RMDBS
   2.非结构化数据(全文数据) doc,xml,html,ppt,txt 文本数据
   *3.半结构化数据 key-value(Redis) ,(MongoDB)Json 文档  ,HBase (NoSQL)
搜索分类:
   1.结构化搜索 SQL structure query language(唯一列,主键)
   2.非结构化搜索 --> 全文数据
      ① 顺序扫描法  特征:简单,效率低下(搜索不可复用),不适用大数据
	  ② 全文检索 : 提取全文数据的部分信息 ,对这些提取的部分信息
					进行归纳总结,将归纳总结出来部分信息称之-->索引
					先创建索引,再针对于索引的搜索 --> 全文检索
					①创建索引
					②索引检索
	特征:复杂,效率高(索引库检索,可复用),适合大数据检索
	     不适合数据 修改 一般只做 只读数据的检索  --->  不可以完全替代数据库
应用领域:电商,旅游,论坛,百度/搜狗/360/google(搜索)
